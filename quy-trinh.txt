TỔNG QUAN
---------
Website bán sản phẩm số (sách PDF, tài liệu) với giao diện dark futuristic.
Tech stack: Vanilla HTML/CSS/JS + Firebase + Vercel + SePay

================================================================================
PHASE 1: SETUP CƠ BẢN
================================================================================

1.1. Tạo project structure
    - Tạo folder gốc
    - git init
    - Tạo .gitignore (node_modules, .env, public/, *.backup)
    - Tạo package.json (npm init)

1.2. Setup Firebase
    - Tạo project trên Firebase Console
    - Bật Firestore Database
    - Bật Firebase Authentication
    - Bật Firebase Storage
    - Lấy Firebase config (apiKey, authDomain, projectId...)
    - Tạo firebase.js (config + export SDK modules)

1.3. Setup Vercel
    - Kết nối GitHub repo với Vercel
    - Tạo vercel.json (buildCommand, outputDirectory, rewrites cho API)
    - Tạo prepare-public.js (script copy files vào public/ khi build)
    - Set environment variables: FIREBASE_SERVICE_ACCOUNT, SEPAY_API_KEY,
      RESEND_API_KEY, INTERNAL_API_KEY

1.4. Tạo file cơ bản
    - favicon.svg
    - firebase.js (Firebase SDK config)
    - auth-ui.js (Login/Register modal)

PHASE 3: DATA LAYER
================================================================================

3.1. Firestore Collections
    - products/{docId} — sản phẩm
    - combos/{docId} — combo ưu đãi
    - blog_posts/{docId} — bài viết
    - orders/{docId} — đơn hàng
    - users/{docId} — khách hàng
    - payments/{docId} — thanh toán
    - emails/{docId} — log email
    - admin_logs/{docId} — log admin
    - site_content/banner — banner thông báo
    - site_content/about — nội dung about
    - site_content/support — nội dung support
    - page_views/{date} — analytics

3.2. Client-side Data (js/firestore-sync.js)
    - SonHaiData.getProducts() — lấy sản phẩm từ Firestore
    - SonHaiData.getCombos() — lấy combo
    - SonHaiData.getBlogPosts() — lấy blog
    - SonHaiData.getBanner() — lấy banner
    - Cache localStorage 5 phút (prefix sndata_)

3.3. Cart (js/cart-badge.js)
    - defaultProducts array (fallback)
    - addProduct() / addCombo() / removeProduct()
    - getCart() / saveCart() / clearCart()
    - Version control (CURRENT_VERSION) để force refresh

================================================================================
PHASE 4: BACKEND API (Vercel Serverless)
================================================================================

4.1. Checkout
    - POST /api/checkout — tạo đơn hàng Firestore

4.2. Thanh toán SePay
    - POST /api/seapay/webhook — nhận webhook từ SePay
    - GET /api/seapay/status — frontend poll trạng thái

4.3. Email (Resend)
    - POST /api/email/send — gửi email qua Resend SDK
    - Templates: order_confirmation (có link tải), welcome, OTP

4.4. Auth (OTP)
    - POST /api/auth/send-otp — gửi mã OTP qua email
    - POST /api/auth/verify-otp — xác thực OTP

4.5. Admin
    - POST /api/admin/confirm-order — xác nhận đơn thủ công

LƯU Ý QUAN TRỌNG
================================================================================

1. Product ID trên Firestore là STRING, không dùng parseInt()
2. So sánh ID luôn dùng String(a) === String(b)
3. getCart() trả về ARRAY, không phải {items: []}
4. Cache localStorage TTL 5 phút, version control để force refresh
5. Ảnh sản phẩm: product/slug_san_pham.webp
6. Email: cần verify domain trên Resend để gửi từ domain riêng
7. SePay webhook cần API key header để xác thực
8. prepare-public.js phải list đầy đủ files cần copy
